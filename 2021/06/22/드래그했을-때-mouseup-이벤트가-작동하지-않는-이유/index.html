Posted on 2021-06-22 Edited on 2021-06-24 In Project/JaDooCalendar

드래그했을 때 mouseup 이벤트가 작동하지 않는 이유

요소를 잡아 끌고 놓았을 때는 해당 요소의 mouseup 이벤트가 발생하지 않기 때문이다.

마우스 왼쪽 클릭 시

  1. mousedown
  2. mouseup
  3. click 이벤트 순으로 발생한다.

마우스 오른쪽 클릭 시

  1. mousedown
  2. mouseup
  3. contextmenu 이벤트 순으로 발생한다.

마우스 왼쪽 클릭 후 잡아 끄는데 해당 요소가 draggable false면

  1. mousedown
    만 일어난다.
    mouseup과 click은 발생하지 않는다.

마우스 왼쪽 클릭 후 잡아 끄는데 해당 요소가 draggable true면

  1. mousedown
  2. dragend 이벤트 순으로 발생한다.

mouseup 이벤트를 document에 걸어준다.

selectstart 를 꺼줘야하는 건 블록 잡히고 잡아 끌면 dragstart가 시작되므로
근데 해당 요소에서만 꺼주면
다른 요소에서부터 잡아 끌면 블록 잡힌다.

요소를 잡고 드래그하면
블록이 씌워지는 문제가

드래그할 버튼을 따로 뺀다.

child 와 children 차이 -> TIL

DocumentOrShadowRoot.activeElement

pointer-event none을 주니 이걸 확인할 필요가 없음.

요소가 따라다니면, 그 요소에 mouseup이벤트를 걸어주면 되었다.

rect으로 계산할 수도 있었지만,
closet를 쓰는 게 나았음

rect로 하면 반정도 내려갔을 때 밑으로 보낸다 같은 게 되지만
글도 길어도 5줄 이상은 textoverflow 처리를 해줄 것이기때문에

drag and drop API는 다른 웹사이트로도 가지고 갈 수 있어 문제가 있다.

.dragging {
opacity: 0.5;
pointer-events: none; // 드래그 할 땐

.item-util {
display: block;
opacity: 1;
}
}

opactiy는 상속이 안된다….

이건 각 웹사이트 마다 주소창 옆 대표 상징 로고들이 있는데 그것을 Favicon 이라고 한다.

CSs에서도 textoverflow가 가능하지만
text-overflow 효과를 적용하려면 white-space:nowrap; overflow: hidden; 값이 있어야 하기 때문에
5줄이라는 제한을 두고 싶은거라 js를 사용하였다.

여러명이서 쓰니까 내가 null 해버리면 그 이벤트에 어떤 이벤트가 걸려있을 지 모르는데 문제가 됨.
$calendarDates.addEventListener(‘mousemover’,mouseoverEventFunction)
$calendarDates.onmousemove = e => {
draggable.style.transform = translate3d(${ e.clientX - initialMousePos.x }px, ${e.clientY - initialMousePos.y}px, 0);
};
